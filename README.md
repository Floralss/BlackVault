# Black Social

Готовая сборка соцсети на HTML/CSS/JS + Firebase (Auth, Firestore, Storage).

## Структура

```
black-social/
├─ index.html          вход / регистрация (почта или телефон)
├─ app.html             основной интерфейс (чаты, поиск, профиль, премиум, настройки, админка)
├─ css/style.css        вся тема оформления (дизайн-токены в :root)
├─ js/firebase-config.js  ключи Firebase + список UID админов
├─ js/auth.js           регистрация/вход
├─ js/app.js            навигация, список чатов
├─ js/profile.js        аватар, музыка профиля, био
├─ js/search.js         поиск людей/каналов/групп
├─ js/chat.js           переписка + блокировка "только Premium"
├─ js/settings.js       приватность + кастомизация интерфейса
├─ js/premium.js        карточки тарифов (только чтение)
├─ js/admin.js          редактирование тарифов админом
├─ firestore.rules      правила доступа к базе данных
└─ storage.rules        правила доступа к файлам (аватары/музыка)
```

## Запуск

1. Залейте всю папку на любой статический хостинг (Firebase Hosting, Netlify, обычный сервер) — это чистый статический сайт, сборщик не нужен.
2. В консоли Firebase включите:
   - **Authentication** → способы входа: Email/Password и Phone.
   - **Firestore Database** (в режиме production).
   - **Storage**.
3. Задеплойте правила безопасности:
   ```bash
   npm install -g firebase-tools
   firebase login
   firebase deploy --only firestore:rules,storage:rules
   ```
4. Откройте `index.html`, зарегистрируйтесь. Скопируйте свой UID из вкладки Authentication → Users.
5. Вставьте этот UID в **два места**:
   - `js/firebase-config.js` → массив `ADMIN_UIDS` (это только показывает кнопку "Админка" в интерфейсе),
   - `firestore.rules` → функцию `isAdmin()` (это и есть реальная защита — без этого шага админ-панель не будет ничего сохранять).
6. Зайдите в раздел **Premium → Админка** и добавьте тарифы: название, цена, период, список возможностей.

## О безопасности — честно

Вы просили сделать сайт "тяжелее для взлома". Важно понимать, как это работает на практике:

- **apiKey в конфиге Firebase — не секрет.** Firebase так спроектирован: этот ключ виден в devtools на любом сайте, использующем Firebase, и сам по себе прав ни на что не даёт. Прятать или обфусцировать его бессмысленно.
- **Настоящая защита — это Security Rules** (`firestore.rules`, `storage.rules`), потому что они выполняются на сервере Google, а не в браузере пользователя. В этой сборке уже заложено:
  - пользователь не может сам выставить себе `premium: true` из консоли браузера;
  - редактировать тарифы может только UID из списка админов;
  - сообщения в чате может читать только тот, кто состоит в чате;
  - файлы аватара/музыки ограничены по размеру и типу.
- **Для полноценной оплаты Premium** нельзя доверять клиенту — нужен платёжный провайдер (Stripe, ЮKassa, LiqPay и т.д.) + Cloud Function, которая после подтверждения оплаты сама проставит `premium: true`. Прямое включение из браузера (как сейчас в `subscribeToPlan()`) — это заглушка, которую обязательно нужно заменить перед реальным запуском.
- Дополнительно рекомендуется включить **Firebase App Check** — он подтверждает, что запросы идут именно с вашего сайта/приложения, а не из чужого скрипта.
- Список админов лучше со временем перенести с обычного UID-массива на **Custom Claims** — тогда его вообще нельзя будет подделать даже теоретически.

## Что уже реализовано

- Регистрация и вход по email+пароль и по номеру телефона (SMS-код через Firebase Phone Auth).
- Профиль: аватар, био, музыка (загрузка файла с компьютера/телефона, воспроизведение прямо в профиле).
- Поиск людей/каналов/групп по подстроке имени.
- Настройки приватности: скрыть номер телефона, скрыть аватар, разрешить писать только Black Premium.
- Тарифы Black Premium редактируются в собственной админ-панели (цена, период, описание, "топовый" тариф) — без правок кода.
- Кастомизация интерфейса: акцентный цвет и скругление углов, сохраняются в профиле и применяются на любом устройстве.
- Тёмная премиальная тема на дизайн-токенах (CSS-переменные) — легко перекрасить весь сайт, поменяв значения в начале `style.css`.

## Что стоит доделать перед боевым запуском

- Реальный платёжный шлюз для Premium (см. выше).
- Полноценные каналы и группы (сейчас в UI есть заглушка — структура в `firestore.rules` для `channels`/`groups` уже заложена).
- Полнотекстовый поиск (Algolia/Typesense) — текущий поиск ищет по началу имени.
- Импорт контактов телефонной книги (функция `matchContacts()` в `js/search.js` уже готова, её нужно вызвать из мобильного окружения, например через Contact Picker API или Capacitor/Cordova-плагин, если будете оборачивать сайт в приложение).
